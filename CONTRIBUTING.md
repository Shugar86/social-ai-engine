# Как участвовать

Спасибо, что интересуешься проектом! Social AI Engine — персональный инструмент,
но я открыт к идеям, issue и PR, если они делают движок лучше.

---

## Быстрый путь

1. **Форкни** репозиторий.
2. **Создай ветку:**

   ```bash
   git checkout -b feature/короткое-описание
   ```

3. **Сделай изменения** минимальным diff'ом (KISS).
4. **Убедись, что проходят тесты и lint:**

   ```bash
   ruff check .
   mypy .
   pytest
   ```

5. **Открой PR** с понятным описанием: что, зачем, как проверял.

---

## Стиль

- Python 3.11+, type hints обязательны.
- Асинхронные вызовы API — через `httpx` или `asyncio`.
- Конфиги — в YAML; код не хардкодит проектные значения.
- Каждый новый модуль реализует существующий интерфейс из `*/interfaces.py`.
- Не коммить `.env`, токены, ключи, `*.pem`, `__pycache__/`, `node_modules/`.

---

## DoD для PR

- [ ] Код проходит `ruff` и `mypy`.
- [ ] Новый функционал покрыт тестами с моками внешних API.
- [ ] Интерфейсы не изменены без согласования.
- [ ] Документация обновлена, если меняется архитектура или запуск.
- [ ] Секреты не попали в git.

---

## Обсудить сначала

Если идея касается:

- архитектурных изменений,
- нового проекта/клиента,
- медицинской этики или публикации от имени реального человека,

— лучше открыть issue и обсудить до PR.
